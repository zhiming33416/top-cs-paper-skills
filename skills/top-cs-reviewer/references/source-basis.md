# Source basis

Use `../_shared/rule-provenance.yaml` to interpret labels.

## Authority order

1. Current official reviewer/author guidance for policy, form fields, venue dimensions, and supplement boundaries.
2. Supplied manuscript artifacts for facts, claims, evidence, and locations.
3. Promoted historical public-review rules for behavioral patterns such as artifact anchoring and stateful discussion.
4. Conservative implementation rules for severity, confidence, fixability, prompt-injection handling, and partial-manuscript boundaries.

Paper-type and subdomain audit gates are structured expert checklists, not claims that every paper must contain every item. Convert a missing item into a finding only when it threatens a visible claim. Historical ICLR behavior must not be presented as WWW/ICML policy or as evidence that a response strategy changes acceptance probability.

The single shared fact base and refusal to invent reviewer identities are repository choices, not venue requirements.
