# Polishing output format

Return:

1. `Route and revision mode`
2. `Diagnosis`
3. `Revised text`
4. `Material changes`
5. `Revision ledger`, classifying prose, claim-strength, citation, and figure changes
6. `Claim-strength changes`
7. `Terminology decisions` for multi-paragraph work
8. `Evidence and wording risks`
9. `Verification status`, separating bibliographic, claim-entailment, LaTeX, and figure checks
10. `Unresolved inputs`

Explain only changes affecting logic, evidence, terminology, layout, or author intent. Do not produce a token-by-token change log unless requested. For a layout task, include the affected pages, observed defect, source change, and before/after verification.
