# Section moves

Use these move orders only after identifying the section's actual reasoning job.

## Abstract

`problem -> precise gap -> approach -> principal evidence -> bounded implication`. Remove background that does not establish the gap and claims not supported in the main paper.

## Introduction

`problem relevance -> technical bottleneck -> representative solution families -> remaining gap -> present answer -> contribution-evidence preview`. Replace chronological citation lists and repeated motivation with narrowing logic.

## Related work

`approach family -> established capability -> limiting assumption -> distinction relevant to this paper`. Keep comparisons symmetric and citations adjacent to attributed claims.

## Method

`problem contract -> assumptions/notation -> overview -> motivated modules -> objective/procedure -> complexity -> training/inference detail`. Move empirical benefit claims to experiments unless they are formal consequences.

## Experiments

`research question -> protocol -> comparator fairness -> metric -> result -> interpretation -> boundary`. Keep setup, observation, and interpretation distinguishable. Preserve negative and mixed evidence.

## Discussion and limitations

`supported finding -> interpretation -> relation to prior expectation -> alternative explanation -> implication -> boundary`. Convert generic caveats into scope statements tied to specific claims.

## Conclusion

`supported contribution -> decisive evidence -> implication -> boundary`. Remove new evidence, citations, mechanisms, and universal impact language.
