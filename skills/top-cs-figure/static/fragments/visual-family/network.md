# Visual Family: Network

Use for graph structure, dependency diagrams, information flow, citation/knowledge graphs, module interactions, and web-scale topology summaries.

- Choose node-link diagrams only when topology is the message. Use matrices or aggregated summaries for dense graphs.
- State what nodes, edges, weights, directions, and colors mean.
- Keep layout deterministic by fixing a seed or using supplied coordinates.
- Do not infer graph communities, centrality, or causality unless those values are supplied or computed in the script and reported.
- For large graphs, show a representative subgraph only when the sampling rule is explicit.
