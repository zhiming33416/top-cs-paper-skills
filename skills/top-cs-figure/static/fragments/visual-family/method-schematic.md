# Visual Family: Method Schematic

Use for model architecture, pipeline, data flow, training/inference workflow, taxonomy, and system deployment figures.

- Use a schematic to explain components and flow, not to invent evidence.
- Align component names with the terminology ledger and manuscript method section.
- Make arrows encode actual data/control flow. Avoid decorative arrows that imply unsupported dependencies.
- Keep diagram text short enough for final figure size; move long explanations to caption or manuscript prose.
- If the schematic is paired with result panels, reuse colors and labels across the schematic and quantitative panels.
