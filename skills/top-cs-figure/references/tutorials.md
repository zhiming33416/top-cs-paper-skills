# End-to-End Tutorials

## Raw runs to benchmark figure

1. Create a v3 source registry with CSV hash and columns.
2. Map task to x, method to series, raw score to `raw_y`, and reserve `y` for the computed estimate.
3. Declare mean, bootstrap CI, 2,000 samples, confidence 0.95, missing policy, and seed.
4. Use a dot or grouped comparison variant and shared legend.
5. Render and inspect each panel's plotted-data CSV.
6. Confirm the render manifest records source hash, observation count, estimator, CI, and seed.
7. Add caption obligations for estimator, CI, n, and aggregation unit; run QA and audit.

## Complex mosaic

1. Give each panel one evidence job and a stable ID.
2. Define a mosaic and mark the main benchmark as `hero_panel`.
3. Reuse semantic method roles across comparison, trend, and frontier panels.
4. Set all repeated legends to shared and define one metric-direction note.
5. Share colorbars only across matrices with the same scale.
6. Render at the final double-column dimensions.
7. Check geometry, minimum text, whitespace, shared guide coverage, and caption panel mapping.

## Schematic revision

1. Extract the existing node and edge terminology without renaming it.
2. Add lanes and groups only when those boundaries are stated by the method.
3. Preserve node IDs; move coordinates or change routing separately from terminology edits.
4. Render before and after bundles.
5. Run revision audit and inspect added/removed SVG text, panel IDs, colors, and normalized-spec changes.
6. Report structural changes without claiming the method itself changed unless the source spec changed.

## Camera-ready delivery

1. Freeze source hashes and normalized spec.
2. Confirm target width/height in millimeters.
3. Export SVG/PDF/PNG and requested TIFF.
4. Run bundle QA including PDF fonts, PNG/TIFF DPI, crop, editable SVG text, grayscale/CVD, and caption obligations.
5. Run audit and retain render manifest, plotted-data CSVs, and figure-audit JSON with the submission bundle.
