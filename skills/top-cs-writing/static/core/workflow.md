# Writing workflow

1. Identify deliverable, venue, paper type, sections, language, audience, word/page constraint, and available artifacts.
2. Extract a source ledger separating observed results, author interpretations, hypotheses, and missing facts.
3. Build and lock the terminology ledger; surface only collisions or domain-sensitive choices.
4. Write one central argument: problem + contribution + strongest evidence + boundary.
5. Build the argument graph and claim-evidence-boundary table; mark unsupported, contradictory, or appendix-dependent links.
6. Select functional rhetorical moves for each section. For a long section, draft job-labeled topic sentences first and repair their dependency chain before prose.
7. Map each paragraph to one primary job: context, gap, approach, result, comparison, interpretation, implication, or boundary. Assign a reasoning job to each figure, table, theorem, and citation placeholder.
8. For experiment prose, map every research question and supplied result to a claim; never run or imply an unsupplied experiment. For a needed figure, emit the shared figure brief rather than rendering it.
9. For a long section or full manuscript with a material ambiguity, show the central argument, paragraph map, terminology collisions, and at most two high-leverage questions. Continue without a confirmation gate when the source already fixes the framing.
10. Draft English prose from evidence outward. Keep claims close to support, preserve LaTeX and citation anchors, and keep metadata verification distinct from claim entailment.
11. Audit reader workflow, cross-section terminology, claim propagation, artifact placement, anonymity, and venue constraints.
12. Return the package defined in `output-format.md`.

Do not skip source, terminology, argument, or evidence planning merely because the user requests prose immediately. On follow-up, revise only the affected claims or paragraphs unless the correction invalidates the section architecture.
