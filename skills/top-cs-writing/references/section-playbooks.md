# Section playbooks

Load this reference when a one-paragraph section fragment is insufficient for planning or diagnosis.

## Introduction

Build a controlled narrowing sequence: concrete problem, technical bottleneck, representative approach families, remaining capability gap, the paper's answer and boundary, then contribution-evidence pairs. Fail when the introduction uses a literature chronology, repeats motivation, lists components as contributions, or hides the central claim.

## Method

Order definitions before use. State the problem contract and assumptions, then give an overview before module details. For each module, connect motivation, operation, interfaces, objective, and computational consequence. End with the training/inference or algorithmic procedure needed for reproduction.

## Experiments

Start from research questions and map each central claim to a test. Separate common setup from results. For each question, specify protocol, comparator fairness, metric, uncertainty, result, interpretation, and boundary. Tie ablations to component claims, robustness to generality, and efficiency tests to resource claims.

## Related work

Synthesize by approach family, assumption, or capability. Describe what each family establishes, where it stops for the present problem, and how the current work differs. Keep citations adjacent to the exact attributed claim.

## Discussion and limitations

Distinguish observation, interpretation, mechanism hypothesis, implication, and uncertainty. Name concrete conditions limiting data, tasks, populations, assumptions, compute, deployment, or harms. State which claim each limitation narrows.

## Title, abstract, and conclusion

Draft them after the evidence architecture stabilizes. Keep their versions of the central claim consistent but not mechanically identical: title names the contribution, abstract summarizes evidence, and conclusion synthesizes meaning and boundary.
