# `top-cs-writing` 技能

[English](README_EN.md) · [返回仓库首页](../../README.md)

状态：**Beta**

从可核对的研究材料出发，为 WWW、ICLR、ICML 或通用计算机科学会议规划论证、组织章节并起草论文内容。

## 适合用它做什么

- 规划整篇论文的中心主张、证据链和章节结构。
- 起草标题、摘要、引言、相关工作、方法、实验、讨论、局限和结论。
- 将中文研究笔记转换成有证据约束的英文草稿。
- 检查跨章节术语、图表引用和主张强度是否一致。

## 典型请求

- `根据这些实验结果，为 ICLR empirical paper 设计论证和章节大纲。`
- `把这些中文方法笔记整理成英文 Method，不要补造实现细节。`
- `检查摘要、引言贡献列表和结论是否表达了同一组贡献。`

## 你需要提供

目标会议或 `generic` 模式、论文类型、要处理的章节，以及可用的结果、方法说明、图表、引用或作者笔记。材料不完整时应明确指出缺口。

## 产出

论证图、章节计划、段落级写作契约、英文或中文草稿、待补证据清单，以及必要的图件或引用 handoff。

## 边界

不会编造实验、引用、数据、实现细节或会议规则。只需要修改已有句子时应使用 `top-cs-polishing`；需要审稿模拟或回复审稿意见时应使用对应技能。

## 运行和依赖

技能依赖相邻的 `skills/_shared` 和派生 evidence。请按[安装指南](../../INSTALL.md)安装完整目录。基础写作流程不要求额外 Python 包。

## 相关技能

- [`top-cs-polishing`](../top-cs-polishing/README.md)：修订已有文本。
- [`top-cs-reviewer`](../top-cs-reviewer/README.md)：投稿前审计。
- [`top-cs-figure`](../top-cs-figure/README.md)：接收 figure brief 并生产图件。
