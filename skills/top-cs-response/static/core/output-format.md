# Response output format

Return:

1. `Runtime parameters and parsed metadata`
2. `Readiness`
3. `Editor-instruction and reviewer issue matrix`, implemented as an issue board with stable IDs, rounds, duplicate links, priority, status, and transition history
4. `Protocol and character budget`
5. `Response strategy`
6. `Point-by-point English response`
7. `Stress-test rounds and stop decision` when activated
8. `Verified revision ledger`
9. `Figure-change handoff` when activated
10. `Requested package deliverables` such as cover letter or LaTeX files
11. `Unresolved items and risk flags`
12. `Chinese confirmation` when the user works in Chinese

The issue matrix must retain source IDs and distinguish existing evidence, completed work, planned work, and unsupported author notes. Omit package sections that are neither requested nor required.
