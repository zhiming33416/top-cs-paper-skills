# Source basis

Use `../_shared/rule-provenance.yaml` to interpret labels.

- Current response permissions, revision rules, interaction model, and live budgets are `official-policy` only when verified for the named venue/year.
- Stable IDs, artifact anchoring, state preservation, and evidence-based disagreement are partly `corpus-derived` from 25 historical public ICLR flows, with the limits recorded in `review-stats.json`.
- Reviewer wording, author notes, experiments, results, manuscript locations, and completed changes are `author-artifact` facts.
- Readiness states, action labels, editor-email parsing, Chinese alignment, issue matrices, and package assembly are `conservative-implementation` or `output-contract` rules.
- Examples and test fixtures are `synthetic-example` materials and contain no evidence about editorial outcomes.

Never elevate historical behavior into current protocol. Never mark a change completed because an author note merely says it was made.
