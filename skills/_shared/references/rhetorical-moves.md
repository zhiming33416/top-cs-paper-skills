# Functional rhetorical moves

Treat moves as communicative jobs, not sentence templates. Reorder or omit a move only when the paper type or available evidence makes its function unnecessary; record the reason.

## Section moves

| Section | Functional sequence |
|---|---|
| Title | object or problem -> differentiating contribution -> bounded scope when needed |
| Abstract | context/stakes -> gap -> approach -> decisive supplied evidence -> implication/boundary |
| Introduction | problem -> unresolved gap -> contribution logic -> evidence preview -> contribution list -> scope |
| Related work | organize alternatives -> state the comparison dimension -> locate the unresolved distinction without caricature |
| Method | goal/inputs -> overview -> components or derivation -> design rationale -> guarantees/complexity/implementation boundary |
| Experiments | research question -> matched protocol -> result -> disaggregated analysis -> interpretation -> failure/boundary |
| Discussion | synthesize findings -> explain plausible meaning -> consider alternatives -> delimit transfer -> practical/theoretical implication |
| Limitations | name boundary -> explain affected claim -> state observed or plausible consequence -> give a bounded mitigation or open question |
| Conclusion | restate solved problem -> contribution -> strongest supported finding -> scope -> forward implication without new claims |

## Topic-sentence scaffold

Before drafting a long section, write one sentence per planned paragraph and label its job. Read only those sentences in order. Repair missing premises, duplicated jobs, unsupported jumps, and reversed dependencies before expanding prose. The scaffold is planning material, not mandatory output.

For full manuscripts, propagate each central claim through introduction promise, method capability, experiment or proof, discussion interpretation, limitation, and conclusion. A later section may narrow an earlier claim but must not silently strengthen it.

## Optional introduction-twice strategy

Use a disposable framing introduction before detailed evaluation planning when it helps expose intended claims. Rewrite the final introduction from supplied results after the evidence is stable. Do not present this strategy as venue policy or evidence of acceptance, and do not use it when results are absent.
