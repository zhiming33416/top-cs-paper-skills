# Section architecture

Use each section for a distinct reasoning job.

- **Title:** name the problem and contribution without unsupported priority claims.
- **Abstract:** establish problem, precise gap, approach, principal evidence, and bounded implication.
- **Introduction:** move from consequential problem to unresolved gap, explain why prior approaches do not close it, then state contributions and evidence.
- **Related work:** organize by competing solution strategy or assumption; end each group with the remaining distinction, not a bibliography dump.
- **Method:** define the setting and notation, motivate each component, specify the forward process, objective, and computational implications.
- **Experiments:** map each research question to protocol, comparator, metric, result, and interpretation. Separate setup from findings.
- **Discussion:** explain what the evidence changes, where it does not generalize, and what mechanisms remain uncertain.
- **Limitations:** state concrete boundaries and likely failure modes; do not use generic disclaimers.
- **Conclusion:** synthesize the supported contribution without introducing new claims or citations.

Draft title and abstract last even when planning them first.
