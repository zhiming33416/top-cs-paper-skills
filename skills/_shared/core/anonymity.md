# Anonymity and integrity

- Treat submission drafts as double-blind unless the user says camera-ready.
- Inspect author names, affiliations, email addresses, acknowledgements, grant numbers, repository owners, project URLs, PDF metadata, paths, and first-person self-citations.
- Refer to the authors' prior work in the third person when venue policy requires it.
- Do not weaken related-work coverage to preserve anonymity.
- Flag non-anonymous supplemental code, dataset links, demo domains, and document properties.
- Never list an LLM as an author. The human authors remain responsible for every claim and sentence.
- Do not expose confidential review content outside the user's requested output.
