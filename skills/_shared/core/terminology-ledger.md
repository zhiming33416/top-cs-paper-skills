# Terminology ledger

Build a terminology ledger before drafting or revising more than one paragraph. Treat it as the manuscript's single source of truth.

| Field | Record |
|---|---|
| Canonical term | Preferred name, capitalization, hyphenation, and notation. |
| First use | Full form and abbreviation, if any. |
| Variants found | Alternative names already present in the source. |
| Scope | Sections, figures, equations, or code identifiers affected. |
| Decision | `locked`, `author-confirmation-needed`, or `collision`. |

Extract methods, modules, datasets, benchmarks, metrics, symbols, units, tasks, populations, and recurring concepts. Use one name for one concept and do not vary technical terms for stylistic variety. Flag one term used for two concepts or multiple terms used for one concept. Preserve established field nomenclature and source-code identifiers.

When an author changes a locked term, update every affected occurrence and the ledger. Never invent a name to resolve an ambiguity.
