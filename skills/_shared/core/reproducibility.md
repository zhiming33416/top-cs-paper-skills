# Reproducibility checklist

Check whether another researcher can recover the reported result from the paper and supplement.

- Data source, license, splits, exclusions, preprocessing, and leakage controls.
- Model architecture, objective, initialization, hyperparameters, stopping rule, and model selection.
- Baselines implemented fairly with comparable information, tuning budget, and compute.
- Metrics defined with aggregation, uncertainty, statistical tests, and number of runs where applicable.
- Hardware, software versions, runtime, memory, and compute budget when material.
- Ablations tied to design claims; robustness and failure analyses tied to scope claims.
- Theorem assumptions and complete proofs; system experiments and deployment conditions.
- Anonymous artifact availability and instructions consistent with venue policy.

Report missing items by the claim they threaten, not as an unprioritized checklist.
